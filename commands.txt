[C1] Queue table overflows on mobile

Location: QueueManagement.tsx:305 — <table> inside .card
Category: Responsive
Description: The members table has 5 columns with no overflow-x: auto wrapper. On screens narrower than ~600px it will either overflow the viewport or compress columns into unreadable widths.
Impact: Clinic staff on tablets in portrait mode or phones cannot manage the queue. Core operator functionality broken.

[C4] Clickable <li> items are keyboard-inaccessible

Location: Dashboard.tsx:298-321
Category: Accessibility
Description: Each queue item is a <li> with onClick={() => navigate(...)} and cursor: 'pointer'. It has no role="button", no tabIndex={0}, and no keyboard handler (onKeyDown for Enter/Space). Keyboard-only users cannot navigate to queue management.
WCAG: 2.1.1 Keyboard — Level A (blocking)
Impact: Any operator who uses keyboard navigation (common with accessibility needs, or simply power users) cannot access their queues.
Recommendation: Change <li> to <li role="button" tabIndex={0} onKeyDown={e => { if (e.key === 'Enter' || e.key === ' ') navigate(...) }}> or convert to <button> styled as a list item.

Medium-Severity Issues
[M1] QRDisplay — recursive setTimeout not cleaned up on unmount

Location: QRDisplay.tsx:47
Category: Performance
Description: fetchData schedules itself via setTimeout(fetchData, ...) for QR rotation. When the component unmounts (user navigates away), the timeout fires and calls setStatus, setQrData etc. on a dead component. The mounted = false guard in the outer useEffect only prevents the initial call, not the recursive chain.
Impact: React warning ("Can't perform a React state update on an unmounted component"), potential memory leak on long-running kiosk sessions that navigate.
Recommendation: Use a useRef to store the timeout ID and clearTimeout in the cleanup. Better: use AbortController pattern.
Command: /harden QRDisplay
